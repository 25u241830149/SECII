## 二、AI 工具链选择及理由

1、AI 编程助手（辅助代码生成、补全、重构）

a. 候选工具评估：

| 评估维度 | GitHub Copilot | Cursor (IDE) | 通义灵码 |
| --- | --- | --- | --- |
| 功能特性 | 行业标杆，代码补全质量高，支持多文件关联理解 | 基于 VS Code 定制，支持全局代码库检索与重写 | 阿里出品，响应快，擅长业务代码与中文注释 |
| 易用性 | 高，可作为插件无缝集成现有 IDE | 中，需要迁移新编辑器，有适应成本 | 极高，插件安装即用，国内网络稳定 |
| 成本 | 付费，学生可通过 GitHub 学生包免费 | 免费版有限制 | 完全免费 |
| 团队熟悉度 | 高，成员有使用经验 | 较高 | 中等 |

b. 最终选型决策

选择工具：GitHub Copilot

选择理由：代码上下文预测与跨文件理解能力强，团队已有使用经验，可直接集成现有 IDE，零迁移成本，能保障核心编码阶段开发效率。

不选其他工具理由：

Cursor：需要更换编辑器，增加团队协同成本

通义灵码：复杂业务逻辑的上下文理解不如 Copilot

2、AI 对话工具（辅助文档生成、需求分析、架构设计）

a. 候选工具评估：

| 评估维度 | Claude | ChatGPT | DeepSeek |
| --- | --- | --- | --- |
| 功能特性 | 软件工程表现优秀，文档与代码输出完整 | 逻辑能力强，但长文本偶尔省略内容 | 国产模型，上下文窗口大，思考深度强 |
| 易用性 | 较高，支持 Artifacts 预览文档 / 图表 | 高，交互成熟 | 极高，国内直连 |
| 成本 | 基础免费 | 免费版有限制 | 完全免费 |
| 团队熟悉度 | 较高，主力工具 | 较高 | 较高 |

b. 最终选型决策

选择工具：Claude

选择理由：生成长文档、架构文档、需求说明书时内容完整不省略，结构严谨，符合软件工程课程标准，Artifacts 功能便于预览成果。

不选其他工具理由：

ChatGPT：生成长内容时常省略，需要反复纠正

DeepSeek：格式排版与代码细节表现不如Claude

3、AI 代码审查工具（保障代码合入质量）

a. 候选工具评估:

| 评估维度 | CodeRabbit | AI Code Reviewer | SonarQube（+ AI） |
| --- | --- | --- | --- |
| 功能特性 | 专为 PR 设计，自动审查、点评缺陷、识别漏洞 | 轻量开源脚本，基础代码审查 | 传统扫描工具 + AI，侧重规范检查 |
| 易用性 | 极高，安装 App 即可全自动触发 | 较低，需配置 YML 与 API | 最低，需自建服务器 |
| 成本 | 公开仓库免费 | 消耗 API 额度 | 部署维护成本高 |
| 团队熟悉度 | 中 | 低 | 低 |

b. 最终选型决策

选择工具：CodeRabbit

选择理由：无缝接入 GitHub PR 流程，自动审查代码、识别漏洞、生成摘要，满足 AI 协作契约中 “代码必须审查” 的要求。

不选其他工具理由：

开源脚本与 SonarQube 配置复杂，对 10 周周期项目过重，无法多轮对话纠错.

4、AI 测试工具（测试用例设计与生成）

a. 候选工具评估:

| 评估维度 | Claude | GitHub Copilot | CodiumAI |
| --- | --- | --- | --- |
| 功能特性 | 分析边界条件，生成高覆盖率单元测试 | IDE 内快速生成基础用例 | 专门测试生成插件，可视化分析 |
| 易用性 | 高，复制代码 + 提示词即可 | 极高，一键生成 | 中，需学习插件操作 |
| 覆盖深度 | 深，能识别订单状态机等异常分支 | 较浅，只覆盖主流程 | 较深 |

b. 最终选型决策

选择工具：Claude

选择理由：可深度分析业务逻辑，生成高质量、覆盖异常的单元测试，比普通插件更符合工程要求。

不选其他工具理由：

Copilot：用例过于基础，缺乏深度测试

CodiumAI：工具链过于复杂，Claude 已能满足需求

总结

主要使用：GitHub Copilot+ Claude

质量测试：CodeRabbit（自动化代码审查）
