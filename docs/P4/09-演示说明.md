# 演示说明

更新时间：2026-06-06

## 1. 演示目标

本次演示聚焦 P4 要求的核心业务闭环：

1. 用户注册并登录
2. 发布任务
3. 另一用户抢单
4. 订单完成
5. 提交评价
6. 查看消息/信用变化

## 2. 环境准备

### 必需软件

- Java 17
- Maven 3.9+
- Node.js 22+
- PostgreSQL
- Redis

### 推荐方式

如果机器安装了 Docker，直接在仓库根目录执行：

```powershell
docker compose up -d
```

如果机器没有 Docker，则需手工准备：

- PostgreSQL 数据库 `secii_db`
- Redis `localhost:6379`

## 3. 启动步骤

### 后端

```powershell
cd backend/campushub-parent
mvn -pl campushub-bootstrap -am -DskipTests package
java -jar campushub-bootstrap/target/campushub-bootstrap-0.0.1-SNAPSHOT.jar
```

启动后访问：

- 后端：`http://localhost:8080`
- Swagger：`http://localhost:8080/swagger-ui.html`

### 前端

```powershell
cd frontend
npm.cmd install
npm.cmd run dev -- --host 127.0.0.1
```

启动后访问：

- 前端：`http://127.0.0.1:5173/`

## 5. 核心演示路径

### 路径 A：主业务闭环

1. 用用户 A 注册并登录
2. 进入“发布任务”页面创建一条任务
3. 用用户 B 登录并在首页/任务列表中找到该任务
4. 用户 B 抢单
5. 用户 A 在订单详情页确认/完成
6. 用户 A 或用户 B 提交评价
7. 在消息中心和信用页查看联动结果

### 路径 B：异常流程

建议额外演示两个异常场景：

1. 发布者尝试抢自己任务
2. 已完成或关闭任务再次抢单

### 路径 C：管理后台

1. 用管理员账号登录
2. 查看用户管理、任务审核、举报中心、公告管理、统计看板

## 7. 演示时的注意事项

- PowerShell 下前端命令请使用 `npm.cmd`
- 后端非集成测试命令若需现场演示，请使用 `mvn --% ...`
- 不要在同一工作区并行跑两个 Maven 构建
- 若现场机器没有 Docker，提前准备好 PostgreSQL 和 Redis

## 8. 建议现场展示顺序

建议按以下顺序演示，最稳：

1. 登录页和首页
2. 任务发布与任务详情
3. 订单列表与订单详情
4. 评价与信用页
5. 消息中心
6. 管理后台

这样既能覆盖 P0，也能自然带出 P1。
