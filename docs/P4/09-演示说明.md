# 演示说明

更新时间：2026-06-07

## 1. 演示目标

本次 P4 演示聚焦 P0/P1 已实现功能，推荐主路径为：

1. 普通用户注册并登录。
2. 发布校园互助任务。
3. 另一普通用户浏览任务并抢单。
4. 发布者确认并完成订单。
5. 双方提交评价。
6. 查看信用分和消息通知。
7. 管理员查看后台统计、用户、任务、举报和公告能力。

不建议把 P2 论坛和账号注销作为主演示路径，因为当前论坛模块未实现业务接口，账号注销尚未实现“存在未完成订单时禁止注销”的后端校验。

## 3. 启动步骤

启动方式与 README 末尾“快速启动指令”保持一致。

### 3.1 启动数据库和 Redis

在项目根目录执行：

```powershell
docker compose up -d
```

检查：

```powershell
docker compose ps
```

期望看到：

- `campushub-secii-postgres`：healthy
- `campushub-secii-redis`：healthy

如果不用 Docker，需要手工准备：

- PostgreSQL 数据库：`secii_db`
- Redis：`localhost:6379`
- 已执行 `backend/campushub-parent/campushub-bootstrap/src/main/resources/db/schema.sql`

### 3.2 启动后端

```powershell
cd backend/campushub-parent
mvn -pl campushub-bootstrap -am -DskipTests package
java -jar campushub-bootstrap/target/campushub-bootstrap-0.0.1-SNAPSHOT.jar
```

访问地址：

- 后端：`http://localhost:8080`
- Swagger：`http://localhost:8080/swagger-ui.html`

### 3.3 启动前端

另开一个终端：

```powershell
cd frontend
npm.cmd install
npm.cmd run dev -- --host 127.0.0.1
```

访问地址：

- 前端：`http://127.0.0.1:5173/`

说明：Windows PowerShell 下建议使用 `npm.cmd`，避免 `npm.ps1` 执行策略问题。

## 5. 核心演示路径

### 路径 A：P0 主业务闭环

1. 用户 A 注册、登录。
2. 用户 A 进入“发布任务”，创建一条快递代取或学习辅导任务。
3. 用户 B 注册、登录。
4. 用户 B 在首页/任务广场查找任务，进入任务详情。
5. 用户 B 点击抢单。
6. 用户 A 在订单详情中确认接单。
7. 用户 A 完成订单。
8. 双方进入订单详情或评价入口提交评价。
9. 查看用户信用分变化。
10. 查看消息中心中的订单/评价通知。

### 路径 B：P1 补充能力

1. 提交举报：用户在相关入口提交对用户、任务或订单的举报。
2. 管理员登录后台，在举报中心处理举报。
3. 管理员查看数据看板。
4. 管理员发布或维护公告，前台查看最新公告。
5. 订单双方在订单详情中使用聊天面板。

### 路径 C：异常流程

建议演示 2 个异常场景：

1. 发布者尝试抢自己发布的任务，后端 `GrabService` 会拒绝。
2. 同一接单者重复抢同一任务，后端会返回冲突错误。

## 7. 演示注意事项

- 不要把论坛写入或讲成 P4 已实现功能。
- 账号注销页面可以说明为个人中心的扩展入口，但不要作为核心闭环演示；后端尚未校验未完成订单。
- 前端构建的大 chunk 警告不影响演示，但如现场解释 CI 输出，需要说明这是性能优化提示。
- 若数据库已有旧数据或 Docker volume 很久未清理，建议确认 `schema.sql` 已应用到当前库。
- 不要在同一工作区同时运行两个 Maven 构建任务。
- 提交到 GitLab 后，补充最近一次 Pipeline 页面截图或 URL。

## 8. 推荐演示顺序

1. 首页与登录注册。
2. 任务发布、任务详情和抢单。
3. 订单列表、订单详情、确认和完成。
4. 评价与信用分页面。
5. 消息中心与订单聊天。
6. 举报提交与管理员处理。
7. 管理后台统计、用户管理、任务管理、公告管理。
