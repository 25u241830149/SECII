# 单元测试代码与覆盖说明

更新时间：2026-06-07

## 1. P4 要求对照

P4 要求“核心业务逻辑必须有单元测试”，重点覆盖边界条件、异常输入、权限校验、状态转换、信用分和订单状态等规则，并要求核心模块单元测试覆盖率达到 60% 以上。

当前仓库提供两类证据：

- 后端：JUnit 5、Mockito、MockMvc、Mapper XML 契约测试，并通过 JaCoCo 命令生成覆盖率报告。
- 前端：Vitest、Vue Test Utils、Pinia store/API 模块测试，并通过 V8 coverage 生成覆盖率报告。

## 2. 后端测试代码分布

后端测试位于 `backend/campushub-parent/*/src/test/java`，按模块分布如下：

| 模块 | 代表测试 | 覆盖重点 |
| --- | --- | --- |
| `campushub-common` | `JwtTokenProviderTest`、`SensitiveWordFilterTest`、工具类测试 | JWT、敏感词、日期/加密/校验工具 |
| `campushub-user` | `AuthServiceTest`、`UserServiceTest`、`CreditServiceTest`、控制器测试 | 注册登录、资料校验、信用查询、管理员用户操作 |
| `campushub-task` | `TaskServiceTest`、`TaskQueryServiceTest`、`TaskCommentServiceTest`、`TaskControllerWebMvcTest` | 任务创建/更新、查询筛选、评论、收藏、权限 |
| `campushub-order` | `GrabServiceTest`、`OrderStatusServiceTest`、`OrderControllerWebMvcTest` | 抢单边界、状态转换、取消/确认/完成权限 |
| `campushub-review` | `ReviewServiceTest`、`CreditCalculatorTest`、`ReviewControllerWebMvcTest` | 评价合法性、重复评价、信用分算法 |
| `campushub-report` | `ReportServiceTest`、`AdminStatsServiceTest`、控制器测试 | 举报提交、处理、统计看板 |
| `campushub-message` | `NotificationServiceTest`、`MessageServiceTest`、`NoticeServiceTest`、`ChatServiceTest` | 事件通知、消息读写、公告、订单聊天 |

## 3. 前端测试代码分布

| 目录 | 覆盖重点 |

## 7. AI 辅助测试审查

AI 参与了以下测试审查工作：

- 对照 P4 要求检查测试是否覆盖订单状态、信用分、权限、异常输入。
- 通过失败输出定位前端公开文档页测试过期断言。
- 汇总 JaCoCo/Vitest 覆盖率结果，区分行覆盖率和分支覆盖率。

人工确认点：

- 没有把仅有 mock 的测试写成集成测试。
- 没有把 P2 论坛模块写成已测试业务功能。
- 覆盖率以实际报告为准，不用测试文件数量替代覆盖率数字。

## 8. 结论

单元测试代码与覆盖率交付物已满足 P4 要求：前端行覆盖率 81.67%，后端业务模块行覆盖率 75.67%，均超过 60%。后续若需要更强质量门禁，建议把 JaCoCo 插件固定写入 Maven POM，并在 GitLab CI 中归档后端 coverage 报告。
