## 6. 与“覆盖率 >= 60%”要求的关系

当前仓库已经具备较完整的测试分布，但还存在一个现实差异：

- 仓库尚未接入 JaCoCo 或其他覆盖率插件
- 因此本次可以证明“测试代码存在且可执行”，但无法在命令行中直接产出“覆盖率百分比”报告

结论：

- 从测试类数量、模块分布和执行结果看，核心业务模块已经得到系统性覆盖
- 但如果老师要求严格的“>=60% 覆盖率数值证据”，建议人工上传前补做以下任一项：
  - 在 Maven 中接入 JaCoCo 并生成 HTML/XML 报告
  - 使用 IDE 覆盖率工具跑一次并截图
