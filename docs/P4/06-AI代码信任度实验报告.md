# “AI 代码信任度实验”报告

更新时间：2026-06-07

## 1. 实验对象

本次实验选择 P4 验证过程中真实暴露的前端测试修复任务：

- 文件：`frontend/src/test/views/PublicDocumentView.test.ts`
- 场景：执行 `npm.cmd run test:coverage` 时，公开文档页测试出现 2 个失败断言。
- 选择原因：该问题直接影响 P4 “单元测试代码与覆盖率”交付物，且修复点小，适合观察 AI 直出代码能否直接使用。

## 2. 实验 Prompt

```text
前端 Vitest 覆盖率测试失败。失败信息显示：
1. PublicDocumentView 帮助中心目录断言 expected length 10 but got 14。
2. 注册页点击用户协议后，测试期望“平台使用规则与双方责任说明”，但当前页面文本包含“用户服务协议”。

请只修复过期测试断言，不修改业务组件和公开文档内容。
```

## 3. AI 直出结果

AI 给出的补丁只修改两个断言：

```diff
- expect(wrapper.findAll('.help-toc-link')).toHaveLength(10)
+ expect(wrapper.findAll('.help-toc-link')).toHaveLength(14)

- expect(wrapper.text()).toContain('平台使用规则与双方责任说明')
+ expect(wrapper.text()).toContain('用户服务协议')
```

## 4. AI 直出运行结果

第一次运行测试时，修复前结果为：

| 测试文件 | 用例 | 结果 |
| --- | --- | --- |
| 27 | 90 | 88 通过，2 失败 |

将 AI 直出补丁应用后重新执行：

```powershell
cd frontend
npm.cmd run test:coverage
```

结果：

| 测试文件 | 用例 | 结果 |
| --- | --- | --- |
| 27 | 90 | 全部通过 |

覆盖率结果：

| 指标 | 数值 |
| --- | ---: |
| Statements | 79.06% |
| Branches | 68.53% |
| Functions | 72.50% |
| Lines | 81.67% |

## 5. 人工审查记录

| 检查点 | 审查依据 | 结论 |
| --- | --- | --- |
| 是否符合当前实现 | `publicDocuments.ts` 中帮助中心共有 14 个 section | AI 将目录数量改为 14 是正确的 |
| 是否修改了业务行为 | 补丁只改测试文件，未改 `PublicDocumentView.vue`、`RegisterView.vue` 或文档内容 | 未引入业务变更 |
| 是否弱化断言 | 仍检查目录数量、页面关键文案和弹窗打开结果 | 没有把断言改成过宽松匹配 |
| 是否符合 P4 测试交付 | 修复后 coverage 命令通过并生成覆盖率报告 | 可作为测试交付证据 |

## 6. 修复前后对比

| 指标 | AI 直出前 | AI 直出后 + 人工审查 |
| --- | --- | --- |
| 编译是否通过 | 不涉及编译，测试执行失败 | 测试执行通过 |
| 功能是否可运行 | 前端业务未受影响 | 前端业务未受影响 |
| 测试是否通过 | 88/90 通过 | 90/90 通过 |
| 覆盖率是否产出 | 因测试失败未产出完整可信报告 | 成功产出 `frontend/coverage/` |
| 主要问题/修复说明 | 测试断言滞后于公开文档内容 | 将断言更新到当前真实文档结构 |

## 7. 信任度分析

本次 AI 直出补丁可以直接使用，但前提是人工提供了完整失败信息，并且修复范围被严格限定为“只改测试断言”。如果 Prompt 没有限定范围，AI 可能会选择修改业务文案来迎合旧测试，反而破坏当前产品内容。

AI 表现较好的方面：

- 能快速从失败输出定位具体断言。
- 能生成最小补丁，没有额外重构。
- 能把“测试过期”与“业务实现错误”区分开。

仍需人工把控的方面：

- 判断 14 个目录项是否确实来自当前需求，而不是页面重复渲染。
- 判断“用户服务协议”是不是正确业务文案。
- 确认测试没有被弱化到失去验证价值。

## 8. 结论

本次实验说明，AI 对小范围测试修复非常有效，但不能免除人工审查。信任 AI 代码的前提不是“它能生成补丁”，而是团队能用源码和测试结果验证补丁没有改错对象、没有降低断言强度、没有掩盖真实业务问题。
