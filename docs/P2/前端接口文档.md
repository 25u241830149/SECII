# CampusHub 前端接口封装文档

## 1. 文档说明

本文档从前端视角定义接口封装规范，目标是把后端 REST 接口映射为前端可直接调用的 API 方法，统一调用方式、参数结构和使用页面。

---

## 2. 前端接口封装原则

### 2.1 模块划分原则

前端 api 目录与后端模块一一对应：

- user.ts：用户、认证、实名认证
- task.ts：任务发布、查询、收藏
- order.ts：订单流转、历史订单
- review.ts：评价相关
- message.ts：消息、公告、聊天
- forum.ts：论坛帖子、评论、互动
- admin.ts：后台管理和统计

其中首页贴吧式信息流不单独新增后端模块，而是以前端组合视图方式复用 task.ts、user.ts、message.ts 的能力完成：

- task.ts 提供信息流主体数据
- user.ts 提供头像、昵称、信用等级信息
- message.ts 提供右上角消息入口和公告提醒

### 2.2 统一调用规范

- 所有接口通过 request.ts 暴露的 request 实例调用
- 默认返回后端 ApiResponse 中的 data 字段
- Token 由请求拦截器统一注入
- baseURL 通过环境变量 VITE_API_BASE_URL 配置，开发环境默认走 /api
- 分页接口统一接收 page、size 参数
- 查询参数优先通过 params 传递
- 创建和更新操作优先通过 data 传递请求体
- 401 响应由 request.ts 统一清理登录态并携带 redirect 跳转到 /login

### 2.3 request.ts 约定示例

```ts
import axios from 'axios'
import { router } from '@/router'
import { pinia } from '@/stores'
import { useAuthStore } from '@/stores/auth'
import { useUserStore } from '@/stores/user'

const request = axios.create({
  baseURL: import.meta.env.VITE_API_BASE_URL || '/api',
  timeout: 10000,
})

function redirectToLogin() {
  const authStore = useAuthStore(pinia)
  const userStore = useUserStore(pinia)
  const currentRoute = router.currentRoute.value

  authStore.logout()
  userStore.clearProfile()

  if (currentRoute.path !== '/login') {
    void router.replace({
      name: 'login',
      query: { redirect: currentRoute.fullPath },
    })
  }
}

request.interceptors.request.use((config) => {
  const token = localStorage.getItem('campushub_token')
  if (token) {
    config.headers.Authorization = `Bearer ${token}`
  }
  return config
})

request.interceptors.response.use(undefined, (error) => {
  if (error.response?.status === 401) {
    redirectToLogin()
  }
  return Promise.reject(error)
})

export default request
```

## 4. API 模块设计

### 4.1 user.ts

封装方法：

```ts
register(data)
login(data)
getUserProfile(userId)
updateUserProfile(userId, data)
getUserCredit(userId)
getPublicUserInfo(userId)
getUserHome(userId)
submitVerification(data)
cancelAccount(userId)
banUser(userId)
reviewVerification(userId, approved)
```

接口映射：

| 前端方法名 | 方法 | 后端路径 | 主要调用页面 |
| --- | --- | --- | --- |
| register | POST | /api/user/register | 注册页 |
| login | POST | /api/user/login | 登录页 |
| getUserProfile | GET | /api/user/profile | 个人主页、资料页 |
| updateUserProfile | PUT | /api/user/profile | 个人资料页 |
| getUserCredit | GET | /api/user/credit | 个人主页 |
| getPublicUserInfo | GET | /api/user/{userId} | 信息流卡片、任务详情 |
| getUserHome | GET | /api/user/{userId}/home | 个人主页 |
| submitVerification | POST | /api/user/verification/submit | 实名认证页 |
| cancelAccount | DELETE | /api/user/account | 账号设置页 |
| banUser | POST | /api/admin/users/{userId}/ban | 后台用户管理页 |
| reviewVerification | POST | /api/admin/users/{userId}/verify | 后台用户管理页 |

示例：

```ts
export function getUserProfile(userId: number) {
  return request.get<UserInfo>('/user/profile', { params: { userId } })
}
```

组合视图说明：

- 顶部头像区通过 authStore + getUserProfile 控制“默认头像 + 注册/登录”与“用户头像菜单”切换
- 个人主页主数据优先通过 getUserHome 拉取，再按需补充 getUserProfile 和 getUserReviews
- 登录成功后调用 getUserProfile，补全 userStore 中的用户资料
- 后台审核页面通过 reviewVerification 调用实名认证审核接口

## 5. 页面调用说明

### 5.1 登录与初始化

- 登录页调用 login
- 登录成功后写入 authStore 与 userStore
- 登录成功后继续调用 getUserProfile 恢复顶部头像区与个人中心基础资料
- 应用初始化时调用 getUnreadCount、getNotices 作为全局数据预加载入口
- 未登录时顶部头像区展示默认头像与注册/登录入口
- 登录后顶部头像区切换为真实头像菜单，点击进入 /me

### 5.2 任务与订单联动

- 首页信息流与分类页调用 getTaskList
- 任务详情调用 getTaskDetail，并在抢单时调用 grabOrder
- 订单管理页调用 getOrderList
- 订单详情页组合调用 getOrderDetail、getChatMessages、createReview
- 顶部“+”按钮负责跳转到分类化发布页，发布页调用 createTask
- 发布页提交前应做表单校验，提交时展示 loading，并从 authStore 注入 publisherId

### 5.3 用户与认证联动

- 个人中心调用 getUserProfile、updateUserProfile、getUserCredit
- 实名认证页调用 submitVerification
- 公开主页调用 getUserHome 和 getUserReviews
- “我的收藏”页面调用 getFavoriteTasks，并预留论坛收藏结果扩展位
- “我的足迹”页面通过多个接口组合生成聚合时间线，不强制要求单一后端接口

### 5.4 论坛与消息联动

- 首页可并行调用 getNotices、getTaskList、getPostList
- 帖子详情页调用 getPostDetail、createComment、likePost、favoritePost
- 消息中心调用 getMessages 与 markMessageAsRead

### 5.5 首页信息流展示说明

- 首页主列表使用 task.ts 返回的任务数据，但视觉上按贴吧帖子卡片样式展示
- 卡片展示字段包括头像、昵称、信用称号、标题、描述和图片
- 顶部搜索栏驱动 keyword 查询
- 排序切换器驱动 sort = time | hot
- 分类侧边栏驱动 category 参数切换
