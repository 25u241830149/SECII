# CampusHub 校园互助服务平台统一接口文档

## 1. 文档说明

本文档用于统一管理 CampusHub 后端接口，供前后端联调、测试用例编写、接口验收与版本维护使用。

文档覆盖用户、任务、订单、评价、消息、论坛、后台统计等模块的接口定义，统一约束请求路径、请求方式、参数形式和返回结构。

---

## 2. 通用约定

### 2.1 基础路径

- 服务基础前缀：/api
- 数据格式：application/json
- 字符编码：UTF-8

### 2.2 鉴权约定

- 登录、注册接口默认可匿名访问
- 普通业务接口默认需要登录态
- 后台接口默认要求管理员角色
- 需要业务身份标识的接口按接口定义传递 userId、reviewerId 等参数
- 请求头约定：Authorization: Bearer <token>

### 2.3 统一响应格式

所有接口统一返回 ApiResponse<T>，结构如下：

```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

字段说明：

- code：业务状态码，成功固定为 200
- message：响应说明
- data：实际响应数据

错误响应示例：

```json
{
  "code": 500,
  "message": "error message",
  "data": null
}
```

### 2.4 分页响应格式

涉及列表分页的接口，data 字段使用 PageResponse<T>：

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "records": [],
    "total": 0,
    "page": 1,
    "size": 10
  }
}
```

字段说明：

- records：当前页数据列表
- total：总记录数
- page：当前页码
- size：每页条数

### 2.5 分页参数默认值

- page 默认值：1
- size 默认值：10

### 2.6 接口范围

接口范围包括以下路径族：

- /api/user、/api/admin/users
- /api/tasks、/api/admin/tasks
- /api/orders、/api/admin/reports
- /api/reviews
- /api/messages
- /api/posts、/api/admin/posts
- /api/admin/stats

## 3. 用户与认证模块

### 3.1 接口清单

| 接口名称 | 方法 | 路径 | 说明 |
| --- | --- | --- | --- |
| 用户注册 | POST | /api/user/register | 创建用户账号 |
| 用户登录 | POST | /api/user/login | 登录并获取 Token |
| 获取个人资料 | GET | /api/user/profile | 查询用户资料 |
| 更新个人资料 | PUT | /api/user/profile | 更新用户资料 |
| 获取信用分 | GET | /api/user/credit | 查询用户信用分 |
| 获取公开用户信息 | GET | /api/user/{userId} | 获取用户公开资料 |
| 获取个人主页 | GET | /api/user/{userId}/home | 获取主页聚合信息 |
| 提交实名认证 | POST | /api/user/verification/submit | 提交认证材料 |
| 注销账号 | DELETE | /api/user/account | 注销账户 |
| 后台封禁用户 | POST | /api/admin/users/{userId}/ban | 管理员封禁用户 |
| 后台审核认证 | POST | /api/admin/users/{userId}/verify | 管理员审核实名认证 |

### 3.2 关键请求参数

#### 3.2.1 用户注册

路径：POST /api/user/register

请求体：

```json
{
  "username": "alice",
  "password": "123456",
  "email": "alice@example.com",
  "phone": "13800000000",
  "realName": "张三"
}
```

响应数据：null

#### 3.2.2 用户登录

路径：POST /api/user/login

请求体：

```json
{
  "username": "alice",
  "password": "123456"
}
```

响应数据：

```json
{
  "userId": 1,
  "username": "alice",
  "token": "jwt-token",
  "role": "USER"
}
```

#### 3.2.3 获取个人资料

路径：GET /api/user/profile?userId={id}

查询参数：

- userId：用户 ID

响应数据：UserInfoDTO

主要字段：id、username、email、phone、nickname、realName、department、avatar、role、creditScore

#### 3.2.4 更新个人资料

路径：PUT /api/user/profile?userId={id}

请求体：

```json
{
  "email": "alice@example.com",
  "phone": "13800000000",
  "nickname": "小智",
  "realName": "张三",
  "department": "计算机学院",
  "avatar": "https://example.com/avatar.png"
}
```

#### 3.2.5 获取个人主页

路径：GET /api/user/{userId}/home

响应数据：UserHomeDTO

主要字段：userId、nickname、avatar、creditScore、creditLevel、publishedTaskCount、completedOrderCount

#### 3.2.6 提交实名认证

路径：POST /api/user/verification/submit

请求体：

```json
{
  "userId": 1,
  "realName": "张三",
  "studentId": "20240001",
  "college": "计算机学院",
  "documentUrl": "https://example.com/card.png"
}
```

#### 3.2.7 注销账号

路径：DELETE /api/user/account?userId={id}

查询参数：

- userId：用户 ID

#### 3.2.8 后台审核认证

路径：POST /api/admin/users/{userId}/verify?reviewerId={reviewerId}&approved={true|false}&remark=

查询参数：

- reviewerId：审核管理员 ID
- approved：是否通过
- remark：审核备注，可选
