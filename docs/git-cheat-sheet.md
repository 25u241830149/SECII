# Git 常用指令大全 (适合新手与团队协作)

本指南汇总了日常开发中最常用的 Git 指令，按照使用场景进行了分类，非常适合团队在开发“智协”项目时作为速查手册。

---

## 1. 基础配置与初始化

在第一次使用 Git 或者开启一个新项目时使用。

- `git config --global user.name "你的名字"`：全局设置提交代码时的用户名。
- `git config --global user.email "你的邮箱"`：全局设置提交代码时的邮箱。
- `git init`：将当前普通文件夹初始化为一个 Git 本地仓库（会生成一个隐藏的 `.git` 文件夹）。
- `git clone <仓库地址>`：将云端（如 GitHub）的仓库完整下载到本地。

---

## 2. 日常开发工作流（⭐️ 最高频使用）

这三个指令构成了你每天写代码的“保存”循环。

- `git status`：**查看当前状态**（看看哪些文件被修改了，哪些还没暂存，**强烈建议每次敲指令前都查一下**）。
- `git add <文件名>`：将某个文件的修改添加到“暂存区”（准备提交）。
- `git add .`：**暂存所有**修改过或新建的文件（最常用）。
- `git commit -m "提交说明"`：将暂存区的内容正式提交到本地仓库。**提交说明一定要写清楚做了什么**（例如：`git commit -m "完成用户注册功能"`）。

---

## 3. 分支管理（团队协作核心）

根据你们的规范（`main` + `dev` + `feature`），分支指令非常重要。

- `git branch`：查看本地有哪些分支（带 `*` 号的是你当前所在的分支）。
- `git branch -a`：查看包括云端在内的所有分支。
- `git checkout <分支名>`：**切换**到指定分支（例如 `git checkout dev`）。
- `git checkout -b <新分支名>`：**创建并立即切换**到新分支。
  *(开发新功能时：`git checkout -b feature/login`)*
- `git merge <分支名>`：把指定分支的代码**合并**到当前分支。
  *(例如：你当前在 `dev`，运行 `git merge feature/login`，就会把登录功能合并到开发分支。)*
- `git branch -d <分支名>`：安全删除本地某个分支。

---

## 4. 远程仓库同步（与 GitHub 交互）

- `git remote -v`：查看当前连着哪个 GitHub 仓库。
- `git pull origin <分支名>`：**拉取**云端最新代码并合并到本地当前分支（如果别人推送了新代码，你写代码前**一定要先 pull**）。
- `git push origin <分支名>`：**将本地分支推送到远程仓库**（例如推送主分支：`git push origin main`，推送开发分支：`git push origin dev`）。
- `git push -u origin <分支名>`：**【极其重要】如果是第一次将本地新分支推送到云端**，必须加上 `-u` 参数来建立追踪关系（例如：`git push -u origin feature/login`），以后在这个分支上只需敲 `git push` 即可。
- `git push origin --delete <分支名>`：**删除远程仓库上的分支**（当一个功能开发完毕并合并后，可以通过此命令清理云端分支）。

---

## 5. 查看历史与差异

- `git log`：查看过去的提交历史（按 `q` 退出）。
- `git log --oneline`：在一行内精简显示提交历史，更加直观。
- `git diff`：查看尚未暂存的文件具体改了哪些代码。
- `git diff <分支1> <分支2>`：比较两个分支之间的代码差异。

---

## 6. 时光机：撤销与回退（💥 救命指令）

如果你改错了代码或者提错了交，这些指令能救命。

- `git restore <文件名>`：撤销你在工作区对文件的修改（把还没 `git add` 的代码恢复到上一次提交的状态，**危险操作，改动会消失**）。
- `git reset HEAD <文件名>`：如果你已经 `git add` 暂存了文件，但不想提交了，用这个把它移出暂存区（代码修改保留）。
- `git reset --hard HEAD^`：**危险操作！** 回退到上一个版本，且清空自那以后所有的修改。
- `git revert <版本号>`：生成一个新的提交，用来“反做（撤销）”之前某个版本的修改，而不会抹除历史记录（推荐在已推送到云端后使用这种方式）。

---

### 💡 新手协作黄金建议
1. 每天开始写代码前，先执行：`git pull`。
2. 开发新功能，一定要新建分支：`git checkout -b feature/xxx`，不要直接在 `main` 或 `dev` 上乱改。
3. 提交前必看状态：`git status`。
4. 有冲突时不要慌，VS Code 会有友好的图形化界面（合并更改）帮你解决。
