## 概念解释清单
- 运行时刻环境：目标程序运行时用于实现名字、作用域、绑定、数据类型、过程调用、参数传递和系统接口的一组存储与控制机制。
- 逻辑地址：程序可见的地址形式，由操作系统进一步映射到物理地址。
- 物理地址：实际内存空间中的地址，由硬件和操作系统管理。
- 机器字：目标机一次自然处理的数据单位，课件中按 4 字节理解。
- 补白：为满足目标机对齐约束而产生的闲置存储空间。
- 代码区：存放目标程序可执行代码的静态区域，大小在编译时刻确定。
- 静态区：存放编译时大小和位置可确定的数据对象，如全局变量、全局常量和编译器生成数据。
- 动态数据区：运行时才产生和变化的数据区域，主要包括栈区和堆区。
- 静态分配：编译时即可确定对象存储位置和大小的分配方式。
- 动态分配：运行时根据程序执行路径和对象生命期进行的存储分配。
- 栈式存储：与过程调用和返回同步分配、回收的存储方式，适合生命期不超过过程活动的数据。
- 堆存储：用于存放生命期不确定或可能长于创建过程的数据对象。
- 活动：一次过程、函数或方法调用的运行实例。
- 活动树：描述程序一次运行中所有过程活动及调用关系的树。
- 控制栈：管理当前尚未结束的活动记录的运行时栈。
- 活动记录：一次过程活动在控制栈中的存储块，也称栈帧。
- 临时值：表达式求值等过程中产生、无法完全放入寄存器的中间结果。
- 局部数据：活动记录中为当前过程局部变量分配的空间。
- 机器状态：过程调用前需要保存以便返回时恢复的信息，如返回地址和寄存器内容。
- 访问链：在嵌套过程和静态作用域下，用于定位外层过程活动记录的指针链。
- 控制链：指向调用者活动记录的指针，用于恢复调用关系。
- 返回值区：活动记录中存放被调用过程返回结果的区域。
- 实在参数：调用点提供给被调用过程的参数值。
- 调用代码序列：为一次过程调用分配活动记录、传递参数并转移控制的代码段。
- 返回代码序列：恢复机器状态、传递返回值并把控制交回调用者的代码段。
- top：指向实际栈顶的位置，常用于处理变长数据。
- top_sp：指向顶层活动记录定长字段末端的指针，便于用固定偏移访问定长字段。
- eip：指向代码区下一条将要执行指令的指令指针。
- ebp：指向当前栈帧基址的栈底指针。
- esp：指向当前栈顶的栈顶指针。
- 非局部数据：在当前过程外部声明但当前过程可访问的数据。
- 静态作用域：名字绑定由程序文本结构决定的作用域规则。
- 嵌套深度：过程在静态嵌套结构中的层数，可由源程序静态确定。
- 过程指针型参数：以过程作为参数传递时，需要同时携带代码入口和正确访问链的信息。
- 显示表：按嵌套深度保存当前最高活动记录指针的数组，用常量时间访问外层变量。
- 存储管理器：负责堆区空间分配与回收的运行时子系统。
- 空间效率：衡量存储管理器减少堆空间浪费和碎片的能力。
- 程序效率：衡量对象布局和内存层次利用对程序运行速度的影响。
- 分配开销：完成一次分配或回收操作所需的额外时间和维护成本。
- 时间局部性：近期访问过的存储位置很可能很快再次被访问。
- 空间局部性：近期访问过的位置附近的存储位置很可能很快被访问。
- 高速缓存线：主存与高速缓存之间传输数据的连续块单位。
- 页：虚拟内存与主存之间传输和管理的内存块单位。
- 碎片：堆中空闲块与已用块交错导致可用空间被割裂的现象。
- 窗口：堆中的空闲存储块，也称 hole。
- Best-Fit：把请求分配到能容纳它的最小空闲块中的堆分配策略。
- First-Fit：把请求分配到遇到的第一个足够大的空闲块中的堆分配策略。
- 容器：按空闲块大小分类维护的一组空闲块列表。
- 荒野块：可继续向操作系统申请扩展的堆末端空闲块。
- 边界标记：在存储块两端记录空闲/已用状态和块大小的数据结构。
- 嵌入式空闲链表：把空闲链表指针直接存放在空闲块内部的管理方式。
- 内存泄露：程序失去对动态对象的可用引用后仍未释放其空间。
- 悬空指针：指向已被释放对象的指针。
- 解引用：沿指针访问其所指对象的读、写、回收等操作。
- 对象所有者：负责唯一删除某个动态对象的指针或实体。
- 引用计数：为每个对象维护指向它的引用数量，并在数量为 0 时回收对象的方法。
- 区域分配：把生命期相同的一组对象放入同一区域并整体释放的方法。
- 垃圾：狭义指不可达对象，广义指程序不再需要的对象。
- 增变者：用户程序本身，因创建对象和修改引用而改变堆对象关系。
- 垃圾回收：自动识别并回收不可达对象空间的机制。
- 类型安全：运行时能可靠判断数据分量是否为指针及其对象类型的性质。
- 根集：无需指针解引用即可直接访问的数据集合，如静态成员和栈中变量。
- 可达对象：可从根集出发沿引用链访问到的对象。
- 不可达对象：不能从根集沿引用链访问到、可被视为垃圾的对象。
- Stop-the-world：垃圾回收期间暂停用户程序执行的现象。
- 跟踪式回收：从根集出发计算可达对象集合，再回收其补集的 GC 方法。
- 标记-清扫：先标记可达对象，再扫描堆并回收未标记对象的 GC 算法。
- Baker 标记-清扫：用 Free、Unreached、Unscanned、Scanned 四类列表优化清扫范围的标记-清扫算法。
- 标记-压缩：标记可达对象后把它们移动到堆的一端，从而消除碎片的 GC 算法。
- 拷贝回收：把可达对象从一个半空间复制到另一个半空间，并交换半空间角色的 GC 算法。
- 半空间回收器：把堆划分为 fromspace 和 tospace 的典型拷贝回收器。
- 分代回收：根据“多数对象很快死亡”的经验，把堆分成不同代并频繁回收年轻代的策略。
- 记忆集：记录老年代对象到年轻代对象引用的辅助结构。
- Java 强引用：普通引用，只要存在强引用，对象不会被 GC 回收。
- Java 软引用：内存不足时才倾向于被回收的引用，常用于内存敏感缓存。
- Java 弱引用：一旦 GC 扫描到只被弱引用指向的对象，就可回收该对象。
- Java 虚引用：不影响对象生命期，只用于跟踪对象回收活动，必须配合引用队列。
- 引用队列：在引用关联对象即将或已经被回收时接收引用对象的队列。

## 1. 运行时环境与内存组织
- 编译器不仅要翻译源程序，还要在目标机上实现源语言的抽象：名字、作用域、绑定、过程、参数、控制流和数据访问。
- 运行时环境的核心任务是安排数据存储位置、确定变量访问机制、连接过程调用、传递参数和返回值，并提供与操作系统和 I/O 的接口。
- 存储系统可以理解为连续字节块组成的逻辑地址空间，多字节对象占据连续字节，并以第一个字节作为地址。
- 数据对象布局受目标机寻址和对齐约束影响；为满足对齐而浪费的空间称为补白。
- 典型运行时内存分为代码区、静态区、栈区和堆区。
- 代码区和静态区大小通常在编译时刻可确定；栈区和堆区根据程序运行动态变化。
- 为提高空间利用率，栈和堆通常位于剩余地址空间两端并相向增长；实际机器常见栈向低地址增长，课件示例为便于正偏移说明假设栈向高地址增长。

## 2. 静态分配、栈分配与堆分配的适用对象
- 静态分配适用于编译时可确定大小和位置的数据，如全局变量、全局常量、代码区内容和编译器生成的元数据。
- 栈式分配适用于生命期与过程活动同步的数据，如多数局部变量、临时值、参数和返回控制信息。
- 堆分配适用于生命期不确定或可能超过创建过程活动的数据，如 `new` / `malloc` 创建的对象。
- 静态与动态的区别对应编译时刻与运行时刻：静态分配不依赖运行路径，动态分配依赖实际调用、返回和对象创建情况。

### 做题规范 / 通用方法
- 判断对象放置区域时，先问“大小是否编译时可知”，再问“生命期是否超过当前过程活动”。
- 全局量和代码优先归入静态区域；局部且随调用返回消亡的数据归入栈；跨过程存活或手工/自动释放的数据归入堆。
- 若题目给出函数调用图或代码执行过程，应把“何时入栈、何时退栈、何时保留在堆中”分开说明。

## 3. 活动树、控制栈与活动记录
- 一次过程调用称为一个活动；过程活动在时间上具有嵌套性，后调用者先返回。
- 活动树用树形结构描述一次程序运行中全部活动：根为主过程活动，子结点为某活动调用的各个过程活动，兄弟从左到右表示调用先后。
- 调用序列对应活动树的前序遍历；返回序列对应活动树的后序遍历。
- 当前尚未结束的活动正好是活动树中当前结点及其祖先。
- 控制栈保存当前活跃活动的活动记录；栈中活动记录序列对应活动树根到当前活动结点的路径。
- 活动记录可包含临时值、局部数据、机器状态、访问链、控制链、返回值和实在参数。
- 控制链描述动态调用者关系；访问链描述静态嵌套作用域关系，二者用途不同。

### 做题规范 / 通用方法
- 画活动树时，每次调用产生一个新结点，调用谁就作为当前活动的子结点，返回后再考虑右兄弟。
- 判断当前控制栈内容时，从活动树根结点沿路径走到当前活动结点，路径上的活动记录全部在栈中。
- 区分“调用者是谁”和“静态外层是谁”：前者决定控制链，后者决定访问链。

## 4. 调用代码序列、返回代码序列与栈帧布局
- 调用代码序列负责计算实参、分配被调用者活动记录、保存返回地址和必要机器状态，并把控制转移到被调用过程入口。
- 返回代码序列负责放置返回值、恢复 `top_sp` 和寄存器、跳转到返回地址，使调用者从调用点之后继续执行。
- 调用代码序列可分摊在调用者和被调用者之间；为了减少代码重复，通常倾向把更多通用工作放到被调用者中。
- 活动记录布局的一般原则：调用者与被调用者传递的值靠近调用者；控制链、访问链、机器状态等固定长度字段放中间；编译时大小未知的数据放尾部。
- 固定长度字段可通过相对 `top_sp` 的固定偏移访问，变长字段偏移需要运行时计算。
- 调用和返回必须同时满足数据正确性与控制正确性：参数到达被调用者，返回值回到调用者，控制跳入入口并能跳回调用点下一条指令。

### 做题规范 / 通用方法
- 分析调用序列时按“计算实参 -> 保存返回地址和旧栈指针 -> 保存寄存器/机器状态 -> 初始化局部数据 -> 执行被调用者”书写。
- 分析返回序列时按“写返回值 -> 恢复机器状态和栈指针 -> 跳转返回地址 -> 调用者读取返回值”书写。
- 填栈帧图时先标出当前过程的参数、返回值、返回地址、旧基址或旧栈指针，再填局部变量和临时量，最后确定 `SP` 与 `BP` / `top_sp` 位置。

## 5. 栈中变长数据与 `top` / `top_sp`
- 变长数组和运行时大小未知的结构，只要生命期局限于某个过程活动，也可以分配在运行时栈中。
- 这类变长对象位于栈中，但通常不属于活动记录的定长字段部分。
- `top_sp` 用于定位顶层活动记录的定长字段，使固定偏移仍可在编译时确定。
- `top` 指向实际栈顶，即下一个活动记录将开始的位置，随变长对象分配而变化。
- 对变长数据的访问通常先从定长字段中找到指针，再通过运行时计算得到实际地址。

## 6. 运行时环境图解中的寄存器和函数调用过程
- `eip` 指向代码区下一条要执行的指令，函数调用时会跳转到被调用函数代码入口，返回时由返回地址恢复。
- `ebp` 用作当前栈帧基址，便于通过固定偏移访问参数和局部变量。
- `esp` 用作栈顶指针，随压栈、退栈不断变化。
- 程序执行的本质是代码区指令不断改变静态数据区和动态数据区中的状态。
- C 语言常见调用约定中，参数可能按与书写顺序相反的方向入栈；具体题目应以给定栈帧结构和地址增长方向为准。
- 被调用函数返回时，需要恢复调用者栈底和栈顶，并通过 `ret` 或等价机制把返回地址送回指令指针。

### 做题规范 / 通用方法
- 遇到栈图题，先确认题目规定的地址增长方向和栈增长方向，不要套用机器习惯。
- 每执行一次局部变量定义、参数传递、返回地址保存或函数进入/退出，都更新一次 `SP` / `BP` / `eip`。
- 返回过程要同时恢复“数据现场”和“控制现场”：前者是栈帧指针与局部空间，后者是返回地址与下一条指令。

## 7. 非局部数据访问、访问链与显示表
- 没有嵌套过程时，局部变量可用当前活动记录加固定偏移访问，全局变量可用静态区地址访问，函数作为参数传递时通常只需代码入口地址。
- 有嵌套过程且遵循静态作用域时，非局部名字的声明归属可静态确定，但运行时必须找到对应外层过程的活动记录。
- 嵌套深度从外到内递增；不嵌套于其他过程的过程深度为 1，嵌套在深度 `i` 过程中的过程深度为 `i + 1`。
- 访问链规则：若过程 `p` 在静态声明上嵌套于过程 `q`，则 `p` 的活动记录访问链指向栈中最高的 `q` 活动记录。
- 若深度为 `np` 的过程访问深度为 `nq` 的过程所声明变量，则沿访问链前进 `np - nq` 次，再用该变量在目标活动记录中的编译时偏移定位。
- 直接调用时访问链维护分三类：被调用者更深则指向当前活动记录；递归调用则复制当前记录的访问链；被调用者深度小于或等于调用者时，指向其直接外层过程在栈中最高的活动记录。
- 过程指针型参数必须携带代码指针和正确访问链，否则过程执行时无法按静态作用域访问非局部变量。
- 显示表 `d` 为每个嵌套深度保存一个活动记录指针，`d[i]` 指向栈中最高的深度为 `i` 的活动记录。
- 显示表可把外层变量访问开销降为常量，但过程调用和返回时需要保存并恢复相应深度的旧表项。

### 做题规范 / 通用方法
- 访问链题先标每个过程的嵌套深度，再标变量声明所在深度，最后计算 `np - nq`。
- 画访问链时不要按动态调用路径简单连线，要按“被调用过程的静态外层活动记录”连线。
- 显示表题只需判断目标变量声明深度 `i`，再从 `d[i]` 找活动记录；维护题要写出调用时保存旧 `d[np]`、设置新 `d[np]`，返回时恢复旧值。

## 8. 堆管理器、碎片与空闲块组织
- 堆空间用于存放生命周期不确定或直到显式删除才结束的数据对象。
- 存储管理器负责分配连续且大小合适的堆空间，并把回收空间放回空闲空间池，通常不立即归还操作系统。
- 衡量堆管理器要看空间效率、程序效率和低开销：既要减少碎片，也要利用缓存和页的局部性，还要让分配/回收操作快。
- 程序的时间局部性和空间局部性决定了对象布局对缓存性能的重要性。
- 堆碎片来自已分配块和空闲块交错；分配常会切割空闲块，回收则应尽量接合相邻空闲块。
- Best-Fit 保留较大空闲块，但查找成本可能较高；First-Fit 查找快且同一时段对象较可能连续放置，但总体碎片表现可能较差。
- 容器法按块大小维护空闲块集合，小块规格更密，大块可按对数范围分类；小请求直接找对应容器，大请求找合适容器，必要时分割块或使用荒野块。
- 边界标记在块两端记录 free/used 位和块大小，使回收时能检查左右相邻块并接合。
- 嵌入式双向空闲链表把链表指针放在空闲块内部；链表前驱后继表示空闲链表关系，不一定是物理相邻块。

### 做题规范 / 通用方法
- 分配策略比较题按“选择哪个空闲块、优点、缺点、对局部性/碎片的影响”回答。
- 空闲块接合题先查相邻块 free/used 位，再按边界标记得到块大小，最后更新空闲链表。
- 看到容器法题，先按请求大小定位规格容器；若无合适块，再说明分割更大块或从荒野块扩展。

## 9. 手工存储管理的问题与编程模式
- 手工堆管理的两大核心错误是内存泄露和悬空指针引用。
- 空指针解引用和数组越界访问也是非法地址访问问题，可能破坏运行时安全。
- 对象所有者模式要求每个对象任一时刻只有一个 owner，只有 owner 能删除对象；owner 消亡时对象要么被删除，要么所有权已转移。
- 对象所有者模式可防止内存泄露和重复删除，但不能完全解决悬空指针问题。
- 引用计数在赋值、返回和参数传递时维护计数，计数为 0 时删除对象；它可以减少悬空指针问题，但无法回收不可达的循环结构，且每次指针操作都有额外开销。
- 区域分配把生命期相同的对象放在同一区域中，整个区域同时删除，适合生命期结构清晰的对象集合。

## 10. 垃圾回收的基本模型与可达性
- 垃圾回收自动回收不可达对象，适用于 Java、ML、Prolog、Smalltalk 等强调自动内存管理的语言。
- 自动 GC 的基本前提是类型安全：回收器必须知道对象大小，以及哪些字段是指针。
- 根集是不需解引用即可访问的数据，如 Java 静态成员和栈中变量。
- 可达性递归定义：根集对象可达；可达对象字段或数组元素引用的对象也可达。
- 一旦对象变得不可达，它不会自行重新可达，除非程序模型中存在未定义或不安全指针行为。
- 改变可达集合的操作包括对象分配、参数传递、返回值传递、引用赋值和过程返回。
- 引用赋值 `u = v` 可能让 `u` 原来指向的对象失去引用，从而递归地产生更多不可达对象。
- 过程返回会使活动记录出栈、局部变量消失、根集变小，因此可能产生垃圾。
- GC 性能目标包括总运行时间低、空间利用高、停顿时间短和改善局部性。

### 做题规范 / 通用方法
- 可达性判断题从根集出发做图遍历，凡能沿引用边到达的对象标为可达，其余为不可达。
- 引用赋值题先处理左值原引用的丢失，再处理右值引用的复制，最后递归判断是否有对象失去可达路径。
- 过程返回题要先删除当前栈帧中的局部根，再重新从剩余根集计算可达对象。

## 11. 引用计数与跟踪式垃圾回收
- 引用计数在对象分配时设为 1，参数传递时加 1，引用赋值 `u = v` 时对 `u` 原对象减 1、对 `v` 所指对象加 1，过程返回时局部变量所指对象减 1。
- 当对象引用计数变为 0 时，释放它之前要先处理该对象内部指针，使其指向对象的引用计数也相应减少。
- 引用计数不会造成集中停顿，但维护成本高，并且不能发现不可达的循环引用结构。
- 跟踪式回收不在垃圾产生瞬间回收，而是在空闲空间耗尽或低于阈值时周期性运行。
- 跟踪式算法从根集计算可达对象，再回收不可达对象，即回收可达集合的补集。
- 跟踪式算法常用状态包括空闲、未被访问、待扫描和已扫描。

### 做题规范 / 通用方法
- 引用计数题每一步都写清“哪个对象 +1、哪个对象 -1”，减到 0 后立即考虑释放，并递归处理内部指针。
- 跟踪式 GC 题按“根集 -> 标记/扫描可达对象 -> 回收未达对象”组织答案，不需要模拟对象变成垃圾的瞬间。

## 12. 标记-清扫、Baker 优化、标记-压缩与拷贝回收
- 标记-清扫分为标记和清扫两阶段：标记阶段从根集遍历对象引用图并标记可达对象，清扫阶段遍历堆并回收未标记对象。
- 标记-清扫输入可理解为根集、堆和空闲列表 `Free`，输出为删除垃圾后的更新 `Free` 列表。
- 标记-清扫的主要缺点是 Stop-the-world、堆碎片、局部性变差，并且清扫时间与堆大小成正比。
- 若堆大小为 `H`、可达数据量为 `R`，一次标记-清扫代价可表示为 `aR + bH`，回收每个可用字空间的平均代价为 `(aR + bH) / (H - R)`。
- Baker 优化用已分配对象列表避免扫描整个堆；不可达对象等于已分配对象减去可达对象，代价是需要维护该列表。
- Baker 算法用 `Free`、`Unreached`、`Unscanned`、`Scanned` 四个列表表示对象状态并推动标记过程。
- 标记-压缩在标记后计算可达对象新位置，把可达对象移动到堆的一端，并更新所有引用，从而消除碎片。
- 标记-压缩三步为：标记可达对象、为每个可达对象计算 `NewLocation`、移动对象并把引用改为新地址。
- 拷贝回收把堆分为 `fromspace` 和 `tospace`，程序只在 `fromspace` 分配；GC 时把可达对象复制到 `tospace`，结束后交换两个半空间角色。
- 拷贝回收不逐个释放不可达对象，而是搬走可达对象，剩余空间自然成为可用空间。
- 拷贝回收能解决碎片并提升分配效率，但需要保留额外半空间，属于空间换时间。

### 做题规范 / 通用方法
- 标记-清扫题按“从根集标记 -> 清扫整个堆 -> 未标记入空闲表 -> 清除标记位”回答。
- Baker 优化题要写出四个列表的状态迁移：根对象由 `Unreached` 进入 `Unscanned`，扫描后进 `Scanned`，最终残留在 `Unreached` 的对象是垃圾。
- 标记-压缩题必须写“更新引用”，只写移动对象不完整。
- 拷贝回收题必须说明 `fromspace` / `tospace` 角色交换，以及 `free` 指针指向新半空间剩余空闲区开始位置。

## 13. 分代回收与 Java 引用类型
- 分代回收基于经验规律：很多对象生命期很短，年轻代通常包含更多垃圾，因此应更频繁回收。
- 基本做法是把堆切分为带编号的块，新对象在低编号块创建；低编号块满时优先回收，并将存活对象提升到更高编号块。
- 老年代到年轻代的引用较少，但必须用记忆集记录，否则只扫描年轻代时会漏掉从老年代可达的年轻对象。
- Java 强引用表示普通对象状态；只要强引用存在，对象不会被回收。
- Java 软引用在内存充足时尽量保留，在内存不足、抛出 `OutOfMemoryError` 前倾向回收，适合缓存。
- Java 弱引用比软引用生命期更短，只被弱引用指向的对象在正常 GC 扫描到时即可回收。
- Java 虚引用不影响对象生命期，必须配合 `ReferenceQueue`，主要用于跟踪对象被回收的活动。

### 做题规范 / 通用方法
- 分代回收题先说明“新对象进年轻代、年轻代频繁回收、存活者晋升、记忆集记录老到新的引用”。
- Java 引用类型比较题按“是否影响生命周期、何时被回收、典型用途”三列回答。
- 判断对象是否能回收时，强引用优先级最高；若只有软/弱/虚引用，再根据内存状态和 GC 类型判断。
